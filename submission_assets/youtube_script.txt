🌿 DAILY WELLNESS COMPANION — YOUTUBE DEMO SCRIPT
===================================================
Target Duration: 3 to 4 Minutes (Maximum limit: 5 minutes)
Recommended Setup: Have your Web browser running localhost:3000, VS Code, and Terminal open and ready. Be sure to run `npm run db:seed` beforehand so you have data to show!

---

[0:00 - 0:45] SECTION 1: PROBLEM STATEMENT
------------------------------------------
*VISUAL: Show the home screen of the Daily Wellness Companion running in the browser with the clean Glassmorphism interface.*

SPOKEN LINES:
"Hi everyone! This is the 'Daily Mental Wellness Companion,' our team's submission for the Kaggle AI Agents Capstone Project. 

Mental health tracking apps are valuable, but they fail because of high friction. Filling out rigid number grids day after day feels clinical and tedious, so users abandon them. 

Our solution replaces form-filling with an empathetic conversational interface. Users just chat about their day naturally, and our secure system extracts the clinical data automatically."

[0:45 - 1:30] SECTION 2: WHY AGENTS?
------------------------------------
*VISUAL: Switch to VS Code and show the `src/agents/wellness-agents.ts` file.*

SPOKEN LINES:
"Why did we use AI agents to solve this? Standard conversational AI is too generic, prone to hallucination, and lacks persistent memory.

By using specialized agents, we can give each AI a distinct role. We have an empathic 'Check-In Agent' for conversation, a strict 'Extraction Agent' that parses JSON data, a 'Memory Agent' for saving to the database, and 'Pattern' & 'Nudge' Agents for generating insights. This separation of concerns ensures accuracy, privacy, and clinical reliability that a single LLM prompt cannot achieve."

[1:30 - 2:00] SECTION 3: ARCHITECTURE
-------------------------------------
*VISUAL: Show the System Architecture Diagram (open `architecture_diagram.png` from the submission_assets folder).*

SPOKEN LINES:
"Our architecture follows the Agent Development Kit (ADK) pattern. 
When a user chats, the request passes through our Custom Security Middleware, which checks for distress signals. Then, the Orchestrator Agent takes over, routing the transcript through the specialized agents, which use the Gemini 2.5 Flash API. Finally, the data is securely stored in a Supabase PostgreSQL database via Prisma ORM."

SPOKEN LINES:
"Let's see the demo. I'll tell my companion that I slept 8 hours but I'm worried about exams.

Once the check-in is complete, the multi-agent pipeline triggers. You can see the extracted metrics: Mood, Energy, Sleep, and identified worries. 
To enhance the experience, we built advanced clinical UI tools: an interactive Breathing Pacer for anxiety, a Predictive Mood Forecaster slider, and a Binaural Beats audio synthesizer in the sidebar.

*Action: Click the 'Trends & Weekly Insights' tab.*

If we switch to the Trends tab, we can see how the Pattern Agent analyzes our past check-ins to generate visual charts for mood stability and highlight personalized triggers."

----------------------------------
*VISUAL: Switch to VS Code. Show the `src/mcp/server.ts` file, then show the terminal.*
